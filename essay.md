Good afternoon everyone.
My name is Dmytro Laptii.

I’m currently studying Information Systems and Technologies, so it’s probably no surprise that my talk today is about AI.

We’re living in a time where AI is everywhere.
For me, it’s not just a buzzword — it’s something I use almost every day in my work.

Today, I want to talk about an important shift we’re seeing right now:
moving from using AI just to get things done to using it as a tool for learning.

---

So first, let’s clarify what we mean by “AI” here.

We’re not talking about some sci-fi general intelligence.
We’re talking about generative AI — especially large language models.

You probably know them as ChatGPT, GitHub Copilot, or Cursor.

These tools can generate text, write code, and explain complex ideas.
And because they’re so easy to use, they’re already changing the way we learn.

---

Now, think about the situation today.

A student can get a full solution to almost any problem in seconds.
For example, tasks from platforms like e-olymp — which many of us use —
can simply be pasted into an AI tool, and you get a ready-to-use answer.

But here’s the problem.

It reduces independent thinking.
Learning turns into a “copy-paste” process.
And it becomes much harder for teachers to understand what a student actually knows.

So the key question is:
**Is AI a threat to education, or is it an opportunity?**

---

When we talk about risks, one of the biggest ones is losing depth of understanding.

If you get an answer instantly, you skip the “struggle” —
and that struggle is actually important for learning.

There are also problems with assessment.
How do we know if the work is original?

And over time, a passive mindset can appear:
“Why think about this if I can just ask AI?”

---

Now, let’s look at the IT industry.

Because this is where things get interesting.

In real-world development, AI is already a standard tool.

From my own experience — and also from my wife’s —
we work in different companies, and in both cases
AI is not just allowed — it’s encouraged.

Developers actively use tools like GitHub Copilot or Cursor
to speed up their daily work.

---

But here’s the key difference.

There’s a huge gap between a developer using AI as a tool
and a student using AI to skip the learning process.

AI works well only if you already have a strong foundation.

You need to be able to:

* give clear instructions
* understand the output
* and verify if it’s correct

Some of you may have seen memes about “vibe-coding” and “vibe-debugging”.

The idea is simple:
AI writes code very fast…
and then you spend hours trying to understand what went wrong 😄

And honestly — that’s not just a joke. It’s real.

Without fundamentals, you don’t understand the logic,
you can’t verify the code,
and you definitely don’t know how to fix it when it breaks.

So AI doesn’t replace the developer —
it amplifies them, if used correctly.

---

And we also need to remember that AI has limitations.

It doesn’t replace analytical thinking.
It doesn’t design systems for you.
And it doesn’t make high-level decisions.

Sometimes it gives wrong answers.
Sometimes it gives very shallow ones.

So our role has to change.

We need to move from just receiving answers
to actively working with them —
questioning, checking, and improving them.

In other words, we become not just users, but supervisors.

---

I can see this very clearly in my own work.

Right now, I’m developing an OOP project using Cursor.

The workflow is very efficient:
I write a prompt, and it modifies the code directly in my project.

But it doesn’t do everything for me.

I still need to make sure:

* the business logic is correct
* the database structure makes sense
* and all parts of the system work together

And I’ve noticed a very clear pattern.

If my prompt is clear and structured — AI is extremely helpful.
But if it’s vague — it can introduce mistakes
that are much harder to fix later.

---

So this brings us to the main idea.

Maybe the problem is not AI itself —
but how we use it in learning.

Instead of treating AI as an answer generator,
we should treat it as a learning assistant.

---

One way to do this is the Socratic approach.

Instead of giving direct answers,
AI can ask questions, guide thinking,
and help students move step by step toward a solution.

So instead of
“Here is the answer”
we get
“Let’s think through this together.”

---

And in practice, there are several ways we can do this.

We can configure tools like ChatGPT or Gemini
to give hints instead of answers.

We can create simple custom chatbots or agents
that guide students step by step.

We can use AI for explanations, code reviews,
and analyzing mistakes.

And we can treat AI as a problem-solving partner,
not just a tool for answers.

**All these approaches shift the focus from getting answers
to actually building understanding.**

---

So, to wrap things up:

AI is not a threat — it’s a tool.
The real challenge is how we use it.

And instead of trying to fight these tools,
we should learn how to integrate them into education
in a way that supports critical thinking.

---

Thank you.
